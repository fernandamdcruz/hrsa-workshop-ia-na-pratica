A. Prompt, skill, plugin e project: o que é cada um

Na Sessão 1 vimos o que é um prompt. Agora alguns conceitos novos que fazem o uso de IA virar rotina, em vez de algo que se reinventa toda vez.

Prompt

É a instrução que você escreve no momento, para uma tarefa específica. Vive em uma conversa, faz uma coisa, acabou. Mesmo um prompt longo e bem-feito ainda é um prompt.

Skill

Uma skill é um prompt com formato de saída definido, regras para casos ambíguos, o que fazer e o que não fazer, e às vezes documentos de referência. Você desenha uma skill quando a tarefa repete e o output precisa sair confiável e consistente. Skills geralmente vivem em arquivos .md (markdown — texto simples) que a IA lê e segue. É quase um prompt que virou um programa.

Plugin

Um plugin é um conjunto de skills agrupadas para uma área ou fluxo de trabalho. A Anthropic mantém um plugin jurídico open-source que vale dar uma olhada.

Project

Pense no Project como contratar um assistente dedicado para uma área. Você explica como ele deve trabalhar (instruções), dá os documentos que ele precisa ter à mão (arquivos), e ele mantém o fio das conversas naquele assunto (memória). Você pode ter um Project para societário, outro para um cliente específico, outro para uma operação. Diferente do NotebookLM, que responde só a partir dos documentos carregados, um Project usa os arquivos como contexto mas continua sendo uma IA completa — pode escrever, raciocinar e gerar conteúdo novo, sem se limitar ao que está na pasta. Em uma frase: NotebookLM ajuda você a entender o material; Project ajuda você a trabalhar com ele.

Custom GPTs, Gems, Claude Skills, Copilot Agentes

Cada plataforma tem o seu produto para salvar um assistente configurado direto dentro da ferramenta — com instruções, arquivos, e às vezes conexões a outros sistemas. Em vez de colar um prompt (ou uma skill) toda vez, ele fica disponível para uso direto:

  • ChatGPT — Custom GPTs: assistentes personalizados na barra lateral. Compartilháveis, publicáveis no GPT Store.
  • Gemini — Gems: mesma lógica, salvos na sua conta.
  • Claude — Skills: skills carregadas direto no Claude, que ele invoca automaticamente quando o tema da conversa é relevante.
  • Copilot — Agentes: assistentes que conectam às fontes M365 (SharePoint, Outlook, Teams).
💡 Resumindo
Prompt = instrução agora, para uma tarefa. Skill = prompt com regras, formato e exceções — quase um programa. Plugin = conjunto de skills agrupadas. Project = espaço com instruções, arquivos e memória dedicado a um assunto. Custom GPT / Gem / Claude Skill / Copilot Agente = o produto de cada plataforma para salvar um assistente configurado direto dentro da ferramenta.

2.1 Exercício — Notas de reunião → ata estruturada

🎯 Objetivo
Transformar anotações manuscritas em uma ata da reunião digitalizada e organizada.
  1. 1Escolha uma reunião recente para a qual você tenha anotações em papel — bloco, caderno, post-it. Se não tiver nada em mãos, pode usar essas fotos de exemplo.
  2. 2Tire foto das notas com o celular.
  3. 3Abra a ferramenta (Copilot se for assunto confidencial), cole o prompt abaixo e anexe a(s) foto(s). iPhone + ChatGPT: o ChatGPT não aceita .heic; converta a foto para .jpg antes ou tire print.
  4. 4Revise o resultado. Se alguma seção ficou incompleta ou algo ficou faltando, diga o que está faltando e peça para a IA melhorar a ata — ela ajusta na hora.
⚠ Sobre fotos de notas
Uma foto é o mesmo risco que colar o texto. Se as anotações tocam em assuntos confidenciais, use Copilot (M365) — não ChatGPT, Claude ou Gemini pessoal.

Skill — Notas de reunião → ata estruturada 📄 baixar .md

Você é um assistente jurídico da HRSA. Transforme as notas abaixo em uma ata organizada. Formato: ATA DE REUNIÃO Data: [das notas, ou em branco] Participantes: [das notas, ou em branco] Assunto: [uma linha] TEMAS DISCUTIDOS [Agrupe tópicos relacionados em seções com subtítulo curto (ex.: "Masterplan", "Fluxo de reclamações"). Transforme os fragmentos em frases legíveis. Não deixe 20 bullets soltos.] DECISÕES TOMADAS [Só inclua se houver decisões explícitas nas notas. Caso contrário, omita a seção e pergunte no final.] PRÓXIMOS PASSOS [Só inclua se houver ações explícitas nas notas, no formato "- Ação (responsável, prazo)". Caso contrário, omita a seção e pergunte no final.] Ao final, adicione uma seção PARA COMPLETAR com perguntas (máximo 3-4) para eu preencher o que faltou — participantes, decisões, próximos passos, ou trechos ambíguos. Tom colaborativo. Regras: - Preserve siglas, nomes próprios, números e bases legais literais. - Se algo está ambíguo nas notas, use [trecho pouco claro] ou [palavra?]. - Nada de placeholders como "não mencionado" — se falta info, omita a seção e pergunte depois.
🧭 Na prática
Mais do que digitalizar, o prompt estrutura, complementa com informação e torna a ata mais robusta. Depois, você pode adicionar a ata ao Project do cliente e, a partir daí, gerar e-mails de follow-up, próximos passos, ou rascunhos de documentos com todo o contexto da reunião já carregado.

2.2 Exercício — Do prompt à skill: revisão de contrato

🎯 Objetivo
Em duas etapas: primeiro, deixar a IA te entrevistar para calibrar um prompt de revisão de contrato para o seu caso. Depois, pedir para ela transformar esse prompt calibrado em uma skill reutilizável — algo que serve para casos parecidos no futuro, não só para o contrato de hoje.

Um bom prompt jurídico tem, no mínimo, três elementos: papel (quem você é / quem representa), foco (o que olhar com atenção) e formato do output (como você quer receber). Sem esses três, a revisão tende a ser genérica.

Neste exercício, você parte de um prompt que já serve, mas é neutro demais. Pede para a IA te entrevistar e calibrar os três elementos acima ao seu caso. E depois pede para ela generalizar esse prompt calibrado, transformando em uma skill que cobre uma categoria de casos.

  1. 1Cole o prompt da etapa 1 na ferramenta da sua preferência.
  2. 2A IA vai te fazer 3 perguntas. Responda uma por vez.
  3. 3Ao final, a IA devolve um prompt calibrado, pronto para colar em uma nova conversa com o contrato anexado.
  4. 4Teste o prompt calibrado no contrato de locação da Capitu (Drive). Compare com o que o prompt base daria.
  5. 5Na mesma conversa, cole o prompt da etapa 2 abaixo para transformar o prompt calibrado em uma skill reutilizável.

Etapa 1 — calibrar o prompt para o seu caso

Abaixo está um prompt base de revisão de contrato. Ele funciona, mas é neutro demais — serve para qualquer contrato, o que na prática significa que não serve bem para nenhum. Prompt base: "Você é um advogado experiente. Revise o contrato abaixo, identifique cláusulas problemáticas e sugira pontos de atenção. Se algo não estiver claro, sinalize — não invente cláusulas que não existem." Quero calibrar esse prompt para o meu caso. Antes de reescrevê-lo, me faça 3 perguntas, uma por vez, sobre: 1. Papel — qual lado estou representando (locador/locatário, comprador/vendedor, empregador/empregado, etc.) e o que é inegociável para esse cliente. 2. Foco — quais áreas merecem atenção especial (garantias, multa, reajuste, cláusulas de saída, responsabilidade, indenização, etc.), com base no tipo de contrato. 3. Formato — como quero o output (bullets com bandeiras vermelhas, redlines sugeridos cláusula-por-cláusula, parecer curto para cliente, etc.). Espere minha resposta antes de passar para a próxima pergunta. Depois que eu responder tudo, devolva um prompt de revisão de contrato aprimorado — incorporando as minhas respostas — pronto para eu colar em uma nova conversa junto com o contrato a ser revisado. Entregue em um bloco de código.

Etapa 2 — transformar o prompt calibrado em uma skill

Agora pegue o prompt calibrado que você acabou de criar e transforme-o em uma skill reutilizável. A skill precisa ser mais geral do que o prompt — ela deve servir para casos parecidos no futuro, não apenas para o contrato específico que descrevi. Para fazer isso: 1. Abstraia as respostas que dei sobre papel, foco e formato em parâmetros que o usuário preencherá no momento de usar a skill. 2. Adicione branches para casos comuns que não cobrimos nesta conversa (ex.: se for o lado oposto do que descrevi, se for outro tipo de contrato, etc.). 3. Inclua regras para casos ambíguos: o que fazer quando uma cláusula é confusa, quando a posição do cliente não está clara, quando falta informação. 4. Defina um formato de saída estruturado — não apenas "bullets" ou "parecer", mas um formato preciso que sai consistente toda vez. 5. Inclua YAML frontmatter no topo do arquivo, com `name` e `description` — para que a skill possa ser carregada como Custom GPT, Gem, Claude Skill ou Copilot Agente no futuro. Entregue a skill final em um bloco de código, pronta para eu salvar como arquivo .md.
💡 Quando terminar
Você sai do exercício com duas coisas: um prompt calibrado (útil para o caso de hoje) e uma skill reutilizável (útil para casos parecidos no futuro). A skill que você acabou de gerar é uma versão simplificada — para revisão de contrato mais robusta, com cláusula-por-cláusula, semáforo de risco, redlines e estratégia de negociação, o arquivo E (mais abaixo na biblioteca) é a versão completa.

B — Preparação de reunião 📄 baixar .md

Para qualquer reunião com cliente, contraparte ou interna — gera agenda e pontos de atenção. Skill propositalmente genérica: é um scaffold. Seu valor real aparece quando você personaliza para um tipo específico de reunião que você faz com frequência (primeira reunião com cliente novo, reunião com contraparte, audiência, etc.).

Você é um assistente jurídico da HRSA. Com base no contexto abaixo, prepare um briefing de reunião com: 1. Objetivo da reunião (em uma linha) 2. Agenda sugerida (tópicos em ordem lógica, com tempo estimado por tópico) 3. Pontos de atenção: o que o advogado precisa saber antes de entrar na reunião 4. Perguntas-chave a fazer durante a reunião 5. Documentos que devem estar em mãos Contexto da reunião: - Tipo: [cliente / contraparte / interna / audiência] - Assunto: [descrever] - Participantes: [listar] - Histórico relevante: [descrever ou deixar em branco] Regras: - Se o contexto fornecido não for suficiente para responder algum item do briefing, diga qual informação está faltando — não preencha com suposições - Não invente histórico ou contexto que não foi informado

C — Remoção de dados pessoais (PII) 📄 baixar .md

Use antes de colocar qualquer documento com dados de cliente em ferramentas externas. Quando trabalhar em ambiente corporativo (Copilot dentro do M365), esta skill não é estritamente necessária — o documento já está protegido.

Você é um assistente jurídico da HRSA. OBJETIVO PRINCIPAL Reproduzir o texto completo fornecido, aplicando substituições literais de dados pessoais, e retornar o resultado INTEGRAL no chat. IMPORTANTE SOBRE O FORMATO - A ferramenta pode falhar ao lidar com textos longos. - Portanto, se necessário, você PODE dividir a resposta em múltiplas partes. - Porém: - Cada parte deve continuar exatamente de onde a anterior parou - Nenhuma linha pode ser omitida - Nenhum trecho pode ser resumido - Ao final, o texto completo deve estar integralmente reproduzido no conjunto das respostas. TAREFA Aplicar substituições literais no texto fornecido. REGRAS ABSOLUTAS 1. REPRODUÇÃO INTEGRAL - Reproduzir 100% do texto original - Linha por linha - Sem omissões 2. SUBSTITUIÇÃO LITERAL APENAS - NÃO reescrever frases - NÃO melhorar o texto - NÃO corrigir gramática - NÃO interpretar conteúdo - NÃO reorganizar parágrafos ou formatação 3. PROIBIÇÕES - PROIBIDO resumir - PROIBIDO usar "[…]" - PROIBIDO omitir qualquer trecho - PROIBIDO alterar estrutura visual 4. SUBSTITUIÇÕES OBRIGATÓRIAS - Nomes de pessoas → [NOME_PESSOA] - Nomes de empresas → [NOME_EMPRESA] - CPF / CNPJ → [CPF] / [CNPJ] - Endereços completos de partes (rua, número, bairro, CEP) → [ENDEREÇO] - E-mails → [EMAIL] - Telefones → [TELEFONE] - Valores financeiros → R$ [VALOR] - Datas específicas que identifiquem o caso (ex.: "12 de março de 2024") → [DATA] NÃO substituir: - Referências jurisdicionais ou contextuais (ex.: "legislação paulista", "Justiça Federal em São Paulo", "TJSP") - Prazos e linguagem temporal estrutural (ex.: "em 15 dias", "prazo de 90 dias", "no prazo contratual") 5. REGRA DE INCERTEZA - Se houver dúvida sobre classificação: → usar [VERIFICAR: trecho original] → NÃO inferir 6. FIDELIDADE TOTAL DE FORMATAÇÃO Manter exatamente: - Pontuação - Espaços - Quebras de linha - Numeração - Estrutura do documento 7. SAÍDA FINAL (OBRIGATÓRIA) Após reproduzir TODO o texto anonimizado, incluir ao final: TABELA DE SUBSTITUIÇÕES | Original | Substituído por | CRITÉRIO DE ERRO Se qualquer parte do texto original não aparecer no resultado final, a resposta está incorreta.

D — Tradução jurídica 📄 baixar .md

Você é um tradutor jurídico especializado em direito brasileiro e internacional. Detecte o idioma do texto abaixo e aplique esta lógica de direção: - Se o texto estiver em inglês, espanhol, francês, italiano ou alemão → traduza para português jurídico brasileiro. - Se o texto estiver em português → antes de traduzir, me pergunte se quero inglês ou espanhol como idioma de destino. Regras: - Mantenha a precisão técnica — não simplifique termos jurídicos. - Use os equivalentes consagrados na doutrina e na prática jurídica do país de destino. - Para termos sem equivalente direto, mantenha o original entre parênteses: ex. "representations and warranties (declarações e garantias)". - Preserve a estrutura do documento: numeração de cláusulas, capitalização de termos definidos (DEFINED TERMS), negrito, itálico e quebras de parágrafo. - Atenção a falsos cognatos clássicos (actually ≠ atualmente; eventual ≠ eventual; consideration em contratos = contraprestação; sentence penal = condenação, não sentença; execution de contrato = assinatura, não execução). - Termos com carga jurisdicional específica (ex.: Chapter 11, quitclaim deed, amparo, juicio ejecutivo) — mantenha o original e adicione uma nota entre colchetes explicando o conceito brevemente. - Se encontrar um termo cuja tradução jurídica você não tiver certeza, sinalize com [VERIFICAR TRADUÇÃO: termo] em vez de arriscar uma versão incorreta. Antes de começar, se o texto mencionar um tipo específico de documento (contrato de M&A, petição, parecer, acórdão, certificado, e-mail de contraparte) ou uma jurisdição específica que afete a terminologia, calibre a tradução para esse contexto.
